Create a Process

Navigate to the organisation dashboard. Locate the "Orange Juice Quality Control" process group that we just created and click on the "Add new process" button underneath it.

General Settings#

First we have some general settings to fill out regarding the process that we want to create. Don't worry it's not too bad.

Follow the below table which explains each setting and the values used in our example.

SettingExplanationExample Value
NameThis is the name of the processpH Level
DescriptionThis is the description of the processpH Level taken from 5 samples from every batch
Subgroup sizeSubgroup size means how many samples are measured each time. Use a subgroup size of 1 if every process output will be measured, or use another number if taking a sample quantity. In this example we will take 5 samples from every batch, so the subgroup size is 5. However, if we measured every sample from every batch, then the subgroup size would be 1.5
Unique IDUnique ID is the label given to the subgroups. For example this could be a serial number or a batch code. Don't write the actual number or code here, this is just the label for it.Batch Code
Measurement UnitThis is the unit of measure for the measurements that will be taken. In this example the measurement is pH level. In other examples this may be "millimeters (mm)" or simply a "quantity".pH Level
LSLLSL stands for Lower Specification Limit. This is the lower boundary of what you can accept as good. In this example we can accept a pH between 2 and 4 for the Orange Juice.2
USLUSL stands for Upper Specification Limit. This is the upper boundary of what you can accept as good. In this example we can accept a pH between 2 and 4 for the Orange Juice.4

Phase Settings#

ControlShark allows you to create different "Phases" to track changes in the process over time and see the differences. There is no limit to the amount of phases you can have. The phases bring many advanced use cases for comparing performance and tracking improvements.

In our example we will add a phase called "Baseline" with the description "To understand the starting level of the process before improvement".

You'll need at least one phase to get started!

You'll need at least one phase to start collecting data for your process. A common approach when starting for the first time is to have an initial "Baseline" phase which is designed to measure the current performance of the process before any improvement actions are implemented. This is a good way to track the effectiveness of subsequent process changes by comparing them to the baseline.

Once finished, click Save at the top of the screen and you will be taken to your new process. You won't see any analysis until you add some data, which is coming up next!
